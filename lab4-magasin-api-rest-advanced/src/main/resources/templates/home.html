Architecture DDD

Ce système utilise les principes de Domain-Driven Design pour une architecture robuste et maintenable :

  • Séparation claire des domaines métier
  • Architecture en couches (Domain, Application, Infrastructure, Presentation)
  • Cohérence entre modèles de données et interface utilisateur
  • API REST documentée avec Swagger

Technologies Utilisées

  • Spring Boot 3.5
  • JPA/Hibernate
  • PostgreSQL
  • Thymeleaf
  • Bootstrap 5
  • Docker
  • Maven
  • Java 21
